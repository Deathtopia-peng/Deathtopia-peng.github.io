算法的时间复杂度
发表于 2020-02-26| 更新于 2020-02-26|计算机
阅读量:

算法是问题解决方案的准确而完整的描述。对于同一个问题，运用的算法不一样，在解决问题的过程中耗用的时间就有可能不相同。那么应当如何衡量算法的优劣程度呢？时间复杂度是一种值得参考的衡量标准。在做题中我是没有见到过其他衡量方式的

时间复杂度的定义

假设计算机在执行一条语句时进行一次运算，我们编写如下代码：

c

在这段代码中，实际上我们执行了两条语句，于是计算机进行了两次运算。再观察下面的代码：

c

在这段代码中，我们加入了循环，实际上我们输出了n遍 Hello World! 。这一次，计算机就进行了

1+n+1+n+1=2n+31 + n + 1 + n + 1 = 2n + 3 1+n+1+n+1=2n+3

次运算。

我们引入时间复杂度T(n)T(n)T(n)来描述在算法中操作步骤的重复次数与问题规模的关系。我们一般使用该函数的渐进上界来描述时间复杂度。

假设nnn趋向无穷时，nnn的低次幂相较于高次幂对于函数数值的增加贡献较少，这时我们可以主要研究nnn的高次幂。

综合起来：如果一个算法的执行次数是T(n)T(n)T(n)，那么只保留最高次项，同时忽略最高项的系数后得到函数f(n)f(n)f(n)，此时算法的时间复杂度就是O(f(n))O(f(n))O(f(n))。
